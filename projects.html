Whack-A-Mole

Users can click on randomly generated moles on screen to gain points. Before the main gameplay, there is a game instructions page that the users can view or skip. The system will display the user's score once the timer expires. If the user scored higher than the top three previous scores stored in a database, then their scores, presented alongside their chosen username, will be displayed on the leaderboard. The user will be offered the option to replay or exit.

Key Features

  • Player name input
  • Input validation
  • Error handling
  • Data storage
  • Instructions page
  • Engaging visuals
  • Points tally
  • In-game timer
  • Background music
  • Leaderboard

Diabetes Manager

Up to 300 patients’ data can be recorded and managed. This data includes their surname, diabetes status, and a unique improvement regime (either diet, exercise, or medication). The user can add new patients, update previous existing patients’ records, locate a patient with their surname and view all recorded patients.

Key Features

  • Interactive menu
  • Data storage
  • Input validation
  • Error handling
  • Add new patients
  • Update patient’s diabetes status and improvement regime
  • Locate patients by surname
  • Display all recorded patient details

Taxi Driver Summariser

A taxi driver’s earnings, along with their corresponding London station, can be recorded and tracked over 28 days. This data will be processed to provide a daily and four-week summary. The user can choose between a range of times when earnings were made (morning, afternoon, or evening) and locations (Kings Cross, Liverpool Street, Paddington, and Euston).

Key Features

  • Clear instructions
  • Input validation
  • Error handling
  • Data storage
  • Choice flexibility
  • High level of personalising
  • Accurate calculations
